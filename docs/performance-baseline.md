## 优化建议

### 数据库层

1. **索引优化**: 确保 `customer_name`、`phone`、`created_at` 等高频查询字段有索引
2. **查询优化**: 列表查询避免 `SELECT *`，只查询必要字段
3. **连接池**: 检查 MySQL 连接池配置，避免频繁建连

### 应用层

1. **缓存策略**: 对不常变更的数据（如单位、仓库列表）启用 Redis 缓存
2. **批量操作**: 批量创建场景考虑使用数据库批量插入替代逐条插入
3. **事务粒度**: 检查订单创建事务粒度，避免长事务阻塞

### 并发层

1. **锁策略**: 并发支付场景检查 `order_receivable` 更新是否使用行锁
2. **幂等性**: 支付接口应支持幂等，防止重复扣款
3. **队列化**: 高并发写操作考虑使用消息队列异步处理
